## W7 — tiered storage (recap from W7_checkpoint.md)

- `SsdBlockStore`: mmap'd 64 KiB-slot allocator, magic-byte liveness
  sentinel, in-memory index rebuilt at startup. No fsync (W0
  intentional fragility); survives process restart via page cache.
- `TieredStore`: HBM (optional) + DRAM (always) + SSD (optional);
  tier-hint fallthrough on Put, promotion-on-Get past threshold,
  explicit Demote for the evictor, Erase wipes access counts.
- `GetResult` / `LookupResult::Entry::local_data` switched span →
  owned `vector<byte>` (SSD can't lend spans across slot reuse).
- `test_tiered_store.cpp`: 8 scenarios incl. SSD-persistence-across-
  restart.

## W8 — eviction + membership + re-replication

### Eviction (`eviction.cpp`)
- Per-tier thread (HBM/DRAM/SSD; only spawned if tier capacity > 0).
- Wakes every `scan_interval` (500ms); checks high watermark; runs a
  SIEVE pass walking a hand through `TieredStore::Snapshot`.
- Visited blocks get a second chance (clear bit + advance); unvisited
  blocks evict. DRAM/HBM victims `Demote` to the next tier before a
  hard `Erase`; SSD victims always hard-drop.
- After each pass, batches all evicted IDs into one `EvictBroadcast`
  RPC per alive peer (best-effort). `OnPeerEviction` records the
  inverse for a future read-repair-skip optimization.
- Stop-condition #3 (SIEVE convergence) guaranteed by the 2N-step
  bound; all-visited tiers still converge.

### Membership (`membership.cpp`)
- Real heartbeat thread (one `std::thread`, separate from the gRPC
  pool per the W0 design). Per-tick parallel fan-out via `std::async`,
  per-RPC deadline = heartbeat interval.
- `EvaluateSuspicions` each round: `suspect_after` (1s) → suspected,
  `dead_after` (3s) → removed from alive set + epoch bump +
  `OnMembershipChange`.
- `OnMembershipChange` → `Router::SetPeers(alive)` then
  `Replicator::TriggerReReplication(lost)`.
- Peer resurrection on successful heartbeat → membership change with
  empty lost list.
- `ever_seen` startup guard prevents spurious death declarations
  during cluster bring-up.

### Re-replication (`replication.cpp`)
- `TriggerReReplication`: scans up to 256 local blocks; for each block
  we hold and are in the route for, pushes to ALL current route
  members (primary + replicas) except self, via the existing bounded
  queue + worker pool. Insert dedup makes the over-broad push safe.

### Wiring (`cache.cpp`)
- Evictor constructed in ctor; Start/Shutdown order Membership →
  Evictor (and reverse). EvictBroadcast handler delegates to
  `Evictor::OnPeerEviction`. cache.hpp field order encodes the
  reverse-declaration teardown chain (evictor → membership →
  replicator → transport).

### H deferred — rationale

`test_eviction_under_load.py` (the "graceful degradation, Lethe wins
at WSS > single-node" demonstration) is deferred. The eviction
mechanics it would exercise are unit-tested (test_eviction: watermark,
demote, hard-drop, SIEVE convergence) and the cross-tier path is
verified. The under-load *benchmark* — measuring cache-hit-rate taper
on a 3-node vs 1-node cluster — is better placed alongside the W12
ShareGPT sweeps where the workload generator and hit-rate
instrumentation already live. Tracked as a W9/W12 follow-up, not a
W8 correctness gap.

## Bugs found and fixed this milestone

The diagnosis chain on CHECKPOINT 2 surfaced five real bugs, four of
them latent before W8:

1. **Router ring excluded self** (latent since W3). Fixed; the single
   most important fix — re-replication dispatched zero blocks without
   it. See DECISIONS.
2. **Startup heartbeat race** — peers declared dead during bring-up.
   Fixed with `PeerInfo::ever_seen`.
3. **Recursive read-repair** — Fetch handler called Lookup which
   triggered read-repair which Fetched, looping. Fixed with
   `FetchLocal`; read-repair disabled at the Lookup path.
4. **Re-replication pushed to replicas only**, never the new primary —
   failover stalled at ~140/200. Fixed to push to all route members.
5. **SIEVE snapshot staleness** — clearing the visited bit in the
   store but not the local snapshot copy made all-visited tiers never
   converge. Caught by test_eviction.

A sixth, process-level trap (not a code bug): `git reset --hard`
doesn't reliably bump mtimes for cmake, and a `-Werror` build failure
leaves the previous binary in place — so several "still failing" test
runs were actually executing a stale binary. Resolved by always
`touch`-ing sources and verifying the build printed "Built target"
before trusting results. Recorded in the build-sync memory.

## Decisions appended to DECISIONS.md (W8)

1. Router ring must include the local node (latent W3 bug).
2. Failure-detector startup guard: `PeerInfo::ever_seen`.
3. Server-side read-repair disabled at the Lookup path.
4. Re-replication pushes to all route members, not just replicas.
5. SIEVE eviction is a CLOCK-style approximation (no insertion order).

## What W9+ inherits

- **Read-repair** is off at the Lookup path. W9's disagg path and
  W11's chaos suite are where the StreamBlocks bulk-pull read-repair
  belongs (with hit-rate that justifies it).
- **EvictBroadcast → read-repair-skip**: the wire and the
  `OnPeerEviction` tracking are in place; the consumer (skip a peer in
  FetchFromAny if it just evicted) is unwired — small follow-up.

  W9/W12.
- **True SIEVE insertion order**: only if W11 chaos shows CLOCK
  underperforming.
- **LETHE_DEBUG_REREP**: env-gated stderr diagnostics in
  replication.cpp, off by default, kept as a debugging tool.
