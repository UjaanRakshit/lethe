# W5-6 closeout — RDMA architected, gRPC ships as the data path

**Status:** Closed via gate-#1 fallback (path 3 of the prompt's three
gate paths). The KvTransport abstraction is now fully applied —
Replicator no longer talks gRPC directly; everything goes through the
abstraction. IbverbsTransport exists as a build-gated stub and is
unreachable at runtime; W12 PACE swaps it in for real ibverbs against
real IB hardware.

## Gate path

**Gate #1 fired at minute 5.** The WSL2 dev kernel
(`5.15.167.4-microsoft-standard-WSL2`) ships without the InfiniBand
subsystem (no `/lib/modules/.../kernel/drivers/infiniband/` directory
at all). `rdma_rxe` can't be modprobed because `ib_core` isn't even
present; `rdma link show` returns "Failed to open NETLINK_RDMA
socket"; `ibverbs-utils` isn't installed and wouldn't matter if it
were. Three independent blockers, each sufficient. Full evidence and
the alternatives-considered analysis is in
`docs/decisions/W5_rdma_fallback.md`.

Gate #2 (time budget) and gate #3 (perf honesty) were therefore not
exercised. The realized cost was under 4 hours of focused work against
the 25-hour cap.

## Decisions appended to DECISIONS.md (this milestone)

1. **KvTransport adds Fetch alongside Send** — symmetric pull primitive,
   neither leaks RDMA into the interface.
2. **`transport_` outlives `replicator_`** — field reorder in cache.hpp
   encodes the worker-join → transport-close ordering rule.
3. **W5-6 ships gRPC as the production data path** — gate #1 firing
   record; the W12 swap is a one-line change in cache.cpp.
